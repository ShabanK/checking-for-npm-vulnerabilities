NPM COMPROMISE SCANNER - QUICK START
=====================================

LINUX/MAC
=========
IMPORTANT: Always use 'bash', not 'sh'!

‚úì CORRECT USAGE:
  sudo bash scan-npm.sh
  sudo ./scan-npm.sh

‚úó WRONG (script will detect and show error):
  sudo sh scan-npm.sh

WHY?
The script uses bash 4+ features (associative arrays) that don't work with sh.
The script now detects this automatically and shows a helpful error message.

QUICK SCAN:
  cd /home/dude/Work/npm-check
  sudo bash scan-npm.sh

CUSTOM SCAN:
  sudo bash scan-npm.sh -d /home -o my-results.json --max-depth 15

HELP:
  bash scan-npm.sh --help


WINDOWS (PowerShell)
====================
IMPORTANT: Run PowerShell as Administrator!

‚úì QUICK SYSTEM-WIDE SCAN (All Drives):
  .\scan-npm.ps1 -AllDrives

‚úì SCAN SPECIFIC DRIVES:
  .\scan-npm.ps1 -Directories C:\,D:\

‚úì SCAN SPECIFIC DIRECTORY:
  .\scan-npm.ps1 -Directories C:\Users\YourName\Projects

‚úì CUSTOM DEPTH:
  .\scan-npm.ps1 -AllDrives -MaxDepth 50

‚úì SKIP GLOBAL NPM PACKAGES:
  .\scan-npm.ps1 -AllDrives -SkipGlobal

‚úì SHOW HELP:
  .\scan-npm.ps1 -Help

AVAILABLE FLAGS:
  -AllDrives           Scan all fixed drives (C:\, D:\, etc.)
  -Directories <path[]> Scan specific directory/directories (comma-separated)
  -MaxDepth <int>      Maximum directory depth (default: 999)
  -SkipGlobal          Skip scanning global npm packages
  -CsvPath <path>      Path to CSV file (default: .\consolidated_iocs.csv)
  -OutputPath <path>   Output JSON file (default: .\npm-scan-results.json)
  -Help                Show help message

DEFAULT BEHAVIOR (no flags):
  If you run .\scan-npm.ps1 without any flags, it will scan ALL fixed drives
  with depth 999. This can take a long time!

PERFORMANCE NOTES:
  - Scanning all drives can take HOURS depending on your system
  - Test with a single directory first: .\scan-npm.ps1 -Directories C:\Users\YourName
  - node_modules directories are excluded from recursive traversal but still scanned
  - Progress updates shown every 50 directories


OUTPUT
======
Results are saved to npm-scan-results.json (or your specified output file)
